Q1 — Write down the multivariate sigmoid function

Short answer: The multivariate generalization of the sigmoid is the softmax.
  • Scalar sigmoid: $\sigma(z) = \dfrac{1}{1+e^{-z}} \in (0,1)$ — binary.
  • Softmax (multiclass), for logits $\mathbf{z}=(z_1,\dots,z_K)$: $$\text{softmax}(\mathbf{z})_i = \frac{e^{z_i}}{\sum_{j=1}^{K} e^{z_j}}$$ Components are in $(0,1)$ and sum to 1; it's shift-invariant (subtract $\max_j z_j$ for numerical stability).
  • For $K=2$ it reduces to the sigmoid: $\text{softmax}(\mathbf z)_1 = \sigma(z_1-z_2)$.
  • Distinct from the elementwise sigmoid $\sigma(z_i)$ (multi-label: independent yes/no per class, does not sum to 1).

📄 Full write-up: Notes_Activation_Functions.html, §1.

Q2 — Why do sigmoid and tanh saturate?

Short answer: Both are bounded squashing functions that compress all of $\mathbb{R}$ into a finite interval. Mapping an infinite domain into a finite range must flatten at the extremes, and a flat region means the derivative → 0. That near-zero gradient is "saturation."
  • Sigmoid: $\sigma'(z)=\sigma(z)(1-\sigma(z))$; max $0.25$ at $z=0$, → 0 as $|z|\to\infty$.
  • Tanh: $\tanh'(z)=1-\tanh^2(z)$; max $1$ at $z=0$, → 0 as $\tanh\to\pm1$. (Better than sigmoid — bigger max gradient, zero-centered — but same flat-tail flaw.)
  • Why it matters: backprop multiplies these small per-layer factors; saturated layers contribute ≈ 0, so gradients vanish in deep nets → early layers stop learning. Motivates ReLU (derivative 1 on active branch) and normalization (keeps pre-activations near 0, the high-gradient region).

📄 Full write-up: Notes_Activation_Functions.html, §2.

Q3 — Compare ReLU and GELU

Short answer: ReLU $=\max(0,z)$ is a hard, cheap gate (gradient 1 or 0) but suffers dead neurons. GELU $=z\,\Phi(z)$ ($\Phi$ = Gaussian CDF) is a smooth, probabilistic gate that passes a small nonzero signal for negative inputs, avoiding dead neurons and giving smoother optimization at higher compute cost. Both avoid positive-side saturation.
ReLUGELU
Formula$\max(0,z)$$z\,\Phi(z)$
SmoothnessHard kink at 0Smooth everywhere
NegativesHard 0Small negative dip → 0

Why transformers use GELU: smoothness + nonzero negative gradient give a small but reliable quality gain that compounds at scale (worth the extra FLOPs). Dying-ReLU symptom: a growing fraction of permanently-zero activations → suspect LR/init, or switch to GELU/LeakyReLU.

Q4 — What is GELU again, what is Φ?

Short answer: GELU is the activation $\text{GELU}(z) = z\cdot\Phi(z)$. The symbol is Φ ("Phi"), not θ (theta) — it's the standard Gaussian CDF.
  • $\Phi(z) = \Pr[Z \le z]$ for $Z\sim\mathcal N(0,1)$ — the probability a standard normal is $\le z$. Ranges 0→1, S-shaped, $\Phi(0)=0.5$. Equivalently $\Phi(z)=\tfrac12\big(1+\operatorname{erf}(z/\sqrt2)\big)$.
  • Soft gate intuition — GELU multiplies $z$ by the probability it should be "kept":
    • large $+z$ → $\Phi\approx1$ → output ≈ $z$ (on)
    • large $-z$ → $\Phi\approx0$ → output ≈ 0 (off)
    • near 0 → $\Phi\approx0.5$ → output ≈ $z/2$ (the smooth part)
  • vs ReLU = a hard step (keep if $z>0$ else 0). GELU's smoothness avoids the kink / dead-neuron problem.

Symbol guide: $\Phi$ (Phi) = Gaussian CDF (the GELU gate); $\varphi$ (lowercase phi) = Gaussian PDF (bell curve, $=\Phi'$); $\theta$ (theta) = unrelated, usually model parameters.

📄 Related: Notes_Activation_Functions.html, §3.

Q5 — Why is GELU z·Φ(z)? (the input × CDF motivation)

Short answer: Every activation is input × gate: $z\cdot g(z)$ with $g\in[0,1]$. ReLU's gate is the hard Heaviside step $\mathbf{1}[z>0]$; GELU's gate is the Gaussian CDF $\Phi(z)$ — the smooth, probabilistic version of that step. A CDF is the natural soft step (monotone 0→1, differentiable).

Why that CDF specifically:

  • Gaussian-smoothed step: convolving the hard step with a Gaussian gives exactly $\Phi$ → GELU = ReLU gate averaged over Gaussian input noise.
  • Stochastic-regularizer derivation (the real motivation): mask the neuron with $m\sim\text{Bernoulli}(\Phi(z))$ (keep-probability depends on the input; bigger inputs kept more often), then take the expectation: $$\mathbb{E}[z\,m] = z\,\Pr[m=1] = z\,\Phi(z) = \text{GELU}(z).$$ So GELU blends dropout-style stochastic gating with the data — the CDF appears because it is the probability of keeping the unit.

Consequences: self-gating (the input gates itself) — same family as Swish/SiLU $z\,\sigma(\beta z)$, with $\text{GELU}(z)\approx z\,\sigma(1.702\,z)$. Derivative $\text{GELU}'(z)=\Phi(z)+z\,\varphi(z)$ → slightly negative for small negative $z$ → non-monotonic, a touch more expressive.

One-liner: "GELU is input × Gaussian-CDF gate — the smooth/probabilistic ReLU. It's the expectation of randomly keeping a neuron with probability $\Phi(z)$, unifying dropout-style regularization with input-dependent gating."

📄 Related: Notes_Activation_Functions.html §3; Q3, Q4 above.

Q6 — Why does the stable sigmoid need np.where?

Short answer: It's not math — the two forms $\frac{1}{1+e^{-z}}$ and $\frac{e^z}{1+e^z}$ are algebraically identical. It's floating-point overflow: you must never compute exp(large positive).
  • Form A 1/(1+exp(-z)): clean for $z\ge 0$; for $z\ll 0$, exp(-z)=inf (overflow).
  • Form B exp(z)/(1+exp(z)): clean for $z<0$; for $z\gg 0$, exp(z)=inf → inf/inf = NaN.
  • Each form blows up on one side. np.where(z>=0, A, B) picks the form whose exponent is $\le 0$ (so exp stays bounded in $(0,1]$). Same function, different rearrangement.

Caveats:

  • np.where evaluates both branches (no short-circuit), so the discarded branch can still emit an overflow warning — the selected value is still correct.
  • A genuine single stable equation: $\sigma(z)=\tfrac12\big(1+\tanh(z/2)\big)$ → 0.5*(1+np.tanh(0.5*z)).
  • In practice use scipy.special.expit / torch.sigmoid / jax.nn.sigmoid (stable internally). The from-scratch branch is to show you understand the overflow issue.

Q7 — Bounded (tanh/sigmoid) vs unbounded (ReLU) activations

Short answer: Bounded activations self-limit their output range; unbounded ones don't — and that drives several practical behaviors.
Bounded (sigmoid, tanh)Unbounded (ReLU, GELU)
Range(0,1) / (−1,1)(0,∞)
Tailssaturate → grad→0no positive saturation → grad ~1
Deep netsvanishing gradientstrains deep nets
Scaleself-limiting (stable)can grow → needs normalization
SparsitydenseReLU → exact zeros (sparse)
Costexp (pricier)max (cheap)
Failuresaturationdead neurons
  • Gradient flow: bounded saturate → vanishing gradients in depth; unbounded keep gradients alive → why deep nets use the ReLU family.
  • Scale: bounded = built-in stability; unbounded = rely on LayerNorm/BatchNorm + residual scaling to control magnitude.
  • Info at extremes: bounded discard magnitude (sigmoid(5)≈sigmoid(50)≈1); unbounded preserve "how big."
  • Sparsity/compute: ReLU = exact zeros + cheap max; bounded = dense + exp.
  • Where used: unbounded → hidden layers (representation, gradient flow); bounded → where the range is the point: gates (LSTM/GRU, GLU — sigmoid = "how much to pass"), probabilities (sigmoid/softmax output), bounded regression (tanh).
  • Zero-centering: tanh zero-centered (cleaner updates) vs sigmoid all-positive (biased / zig-zag).
One-liner: "Bounded self-limit → ideal as gates/probabilities but saturate and vanish gradients in depth; unbounded preserve magnitude and keep gradients alive (deep nets) but need normalization and risk dead neurons. Bounded at gates/outputs, unbounded in hidden layers."

Q8 — Derive ∂L/∂z₂ = ŷ − y (softmax + cross-entropy)

Setup: logits $z_2\in\mathbb{R}^K$, one-hot label $y$ (true class $c$). $$\hat{y}_i = \frac{e^{z_{2,i}}}{\sum_j e^{z_{2,j}}}, \qquad L = -\sum_k y_k\log\hat{y}_k = -\log\hat{y}_c$$

Method 1 — substitute softmax first (elegant):

$$L = -\log\frac{e^{z_{2,c}}}{\sum_j e^{z_{2,j}}} = -z_{2,c} + \log\sum_j e^{z_{2,j}}$$

Differentiate w.r.t. $z_{2,i}$: the first term gives $-y_i$ ($=-1$ at $i=c$, else 0); the second term (log-sum-exp) differentiates to softmax itself, $\dfrac{e^{z_{2,i}}}{\sum_j e^{z_{2,j}}}=\hat{y}_i$. Adding:

$$\frac{\partial L}{\partial z_{2,i}} = \hat{y}_i - y_i \quad\Longrightarrow\quad \frac{\partial L}{\partial z_2} = \hat{y} - y$$

Method 2 — full chain rule: $\dfrac{\partial L}{\partial z_{2,i}} = \sum_k \dfrac{\partial L}{\partial\hat{y}_k}\dfrac{\partial\hat{y}_k}{\partial z_{2,i}}$ with CE gradient $\dfrac{\partial L}{\partial\hat{y}_k}=-\dfrac{y_k}{\hat{y}_k}$ and softmax Jacobian $\dfrac{\partial\hat{y}_k}{\partial z_{2,i}}=\hat{y}_k(\delta_{ki}-\hat{y}_i)$. The $\hat{y}_k$ cancels:

$$-\sum_k y_k(\delta_{ki}-\hat{y}_i) = -y_i + \hat{y}_i\sum_k y_k = \hat{y}_i - y_i$$

using $\sum_k y_k\delta_{ki}=y_i$ and $\sum_k y_k=1$ (where one-hot / normalization enters).

Why it matters: gradient = prediction − truth (same form as logistic regression); clean and numerically stable → frameworks fuse softmax + cross-entropy into one op (no division by $\hat{y}_k$ as in Method 2's intermediate).

Q9 — MLE vs MAP

Short answer: Both pick a single best $\theta$. MLE maximizes the likelihood; MAP maximizes the posterior = likelihood × prior. So MAP = MLE + a prior term.

$$\theta_{\text{MLE}} = \arg\max_\theta \sum_i \log p(x_i\mid\theta) \qquad \theta_{\text{MAP}} = \arg\max_\theta \Big[\sum_i \log p(x_i\mid\theta) + \log p(\theta)\Big]$$

Key insight — a prior is a regularizer:

  • Gaussian prior $\theta\sim\mathcal{N}(0,\sigma^2)$ → $\log p(\theta)=-\frac{1}{2\sigma^2}\|\theta\|^2$ → L2 / weight decay. So L2 reg = MAP with a Gaussian prior.
  • Laplace prior → L1 (sparsity).
  • In deep learning: cross-entropy loss alone = MLE; cross-entropy + weight decay = MAP.

Relationships:

  • MLE = MAP with a flat/uniform prior (the prior term drops out).
  • As data grows, MAP → MLE — the likelihood (a sum over $N$ points) dominates the single fixed prior term, so the prior washes out. Prior matters most with little data.
  • MLE overfits small data; MAP regularizes.
  • Both are point estimates (the mode), not the full posterior — unlike full Bayesian inference, which integrates over $p(\theta\mid D)$.
  • Gotcha: MLE is reparameterization-invariant; MAP is not (the prior density transforms under a change of variables).
One-liner: "MAP = MLE + log-prior; the prior acts as a regularizer (Gaussian → L2/weight decay, Laplace → L1). MLE is MAP with a flat prior, and MAP → MLE as data grows."

📄 Related: Day 1 §5.2 (weight decay); Q8 (CE = MLE).

Q10 — What is the chain rule? (and why a sum)

Short answer: The chain rule differentiates a composition by multiplying local derivatives along the path.
  • Single-variable: if $L=f(u)$ and $u=g(z)$, then $\dfrac{dL}{dz}=\dfrac{dL}{du}\dfrac{du}{dz}$ (the sensitivities multiply).
  • Multivariable: if $z_i$ reaches $L$ through several intermediates $\hat{y}_1,\dots,\hat{y}_K$, multiply along each path and sum over all paths: $\dfrac{\partial L}{\partial z_i}=\sum_k \dfrac{\partial L}{\partial \hat{y}_k}\dfrac{\partial \hat{y}_k}{\partial z_i}$.
  • Why the sum: softmax is coupled — nudging one logit $z_i$ changes every $\hat{y}_k$ — so there are $K$ paths from $z_i$ to $L$, and independent contributions to the same quantity add.
  • Picture: the graph $z_i\to\{\hat{y}_1,\dots,\hat{y}_K\}\to L$; trace every path, multiply local derivatives along it, sum the paths.
  • Backprop = this rule applied across the whole computation graph (multiply along edges, sum over paths, reuse shared sub-results).

Q11 — Concrete example: error = prediction − truth

Multiclass (softmax + CE): cat/dog/bird classifier, logits $z=[2.0,1.0,0.1]$, true = cat → $y=[1,0,0]$. Softmax ($S=e^2+e^1+e^{0.1}=11.21$) → $\hat{y}=[0.659,0.242,0.099]$.

$$\frac{\partial L}{\partial z} = \hat{y} - y = [-0.341,\ +0.242,\ +0.099]$$

  • True class → negative gradient → its logit goes up (step $z\leftarrow z-\eta\nabla$).
  • Wrong classes → positive gradients → their logits go down.
  • Magnitude = how much probability sat in the wrong place; entries sum to 0 ($\sum\hat{y}=\sum y=1$).

Same form across models — each loss paired with its matching output activation:

ModelOutputLossOutput gradient
Linear regression$wx+b$MSE$\hat{y}-y$ (residual; e.g. 250−300 = −50)
Logistic regression$\sigma(z)$BCE$\hat{y}-y$ (e.g. y=1, z=0.5⇒ŷ=0.62 → −0.38)
Softmax$\operatorname{softmax}(z)$CE$\hat{y}-y$ (vector)

Why: these are generalized linear models with their canonical loss↔activation pairing (MSE↔identity, BCE↔sigmoid, CE↔softmax) — that pairing is what collapses the gradient to prediction − target. Mismatch it (MSE on a sigmoid output) and you lose it.

Q12 — How to write the outer product (δ aᵀ)

Definition: the outer product of column vectors $u\in\mathbb{R}^m$, $v\in\mathbb{R}^n$ is the m×n matrix $u v^\top$ with $(u v^\top)_{ij}=u_i v_j$ (every element of $u$ × every element of $v$). Vs the inner/dot product $u^\top v$ = a scalar (needs equal lengths).

Example: $[1,2,3]^\top[4,5] = \begin{bmatrix}4&5\\8&10\\12&15\end{bmatrix}$ (3×1 times 1×2 → 3×2).

Why it's the weight gradient: $\delta_2\in\mathbb{R}^{d_{\text{out}}}$ (output error), $a_1\in\mathbb{R}^{d_{\text{in}}}$ (layer input) → $\delta_2 a_1^\top$ is $d_{\text{out}}\times d_{\text{in}}$ = shape of $W_2$; entry $(\partial L/\partial W_2)_{ij}=\delta_{2,i}\,a_{1,j}$ = (error at output $i$) × (input from $j$).

np.outer(d2, a1)              # or  d2[:, None] * a1[None, :]   (broadcasting)
torch.outer(d2, a1)          # or  d2.unsqueeze(1) @ a1.unsqueeze(0)
jnp.outer(d2, a1)            # einsum: np.einsum('i,j->ij', d2, a1)

Mini-batch: sum outer products over examples = matmul $\Delta A^\top$ ($\Delta$: $d_{\text{out}}\times N$, $A$: $d_{\text{in}}\times N$). The batch dim contracts → backprop is matmuls, not loops.

Q13 — The product-of-Jacobians lens (vanish/explode)

Idea: going backward through $L$ layers, each multiplies the gradient by a factor $r \approx \lVert W\rVert\cdot|\phi'|$, so the gradient at layer 1 is $\approx r^L$. A power → small deviations from 1 compound.
factor $r$$r^{10}$regime
0.50.001vanish fast
0.80.107vanish slow
1.01.0stable
1.12.6explode
1.557.7explode

Every stabilizing trick keeps $r\approx1$:

  • Init (Xavier/He): $\lVert W\rVert\approx1$ → starting factor ~1.
  • Normalization: activations unit-scale → $\phi'$ stays responsive (non-saturated) → factor ~1 during training.
  • Residuals $y=x+F(x)$: Jacobian $I+\partial F/\partial x$ → a literal ×1 path → product can't collapse to 0.
  • Warmup/clipping: stop $r$ transiently spiking >1 early.

Concretely: sigmoid $\phi'\le0.25$ → factor ≤0.25 → vanishes (why deep sigmoid nets won't train); ReLU active $\phi'=1$ + $\lVert W\rVert\approx1$ → factor ~1 → stable.

Q14 — Which part is the Jacobian?

Definition: a Jacobian is the matrix of all first-order partials of a vector function $f:\mathbb{R}^n\to\mathbb{R}^m$ → the $m\times n$ matrix $J_{ij}=\partial f_i/\partial x_j$.
LayerFunctionJacobian
Linear$z=Wx+b$$\partial z/\partial x = W$ → backward "× $W^\top$"
Activation$a=\phi(z)$$\partial a/\partial z = \operatorname{diag}(\phi'(z))$ → backward "⊙ $\phi'$"
Softmax$\hat{y}=\operatorname{softmax}(z)$$\operatorname{diag}(\hat{y}) - \hat{y}\hat{y}^\top$

So the backprop rules are multiplications by (transposed) layer Jacobians:

$$\delta_1 = \underbrace{\operatorname{diag}(\phi'(z_1))}_{\text{activation Jacobian}}\,\underbrace{W_2^\top}_{\text{linear Jacobian}}\,(\hat{y} - y)$$

The chain rule stacks them → the net's input→output derivative is the product of per-layer Jacobians; $r \approx \lVert W\rVert\cdot|\phi'|$ is the magnitude of one. Backprop computes a vector–Jacobian product $v^\top J$ (e.g. $\delta^\top W$, or elementwise $\times\phi'$), never the full matrix.

Q15 — Why constant variance, and the 2 in He init

Why constant variance: $z=Wx$ sums over fan_in inputs; if each layer scales the signal variance by a factor $\neq1$, over $L$ layers it goes like $\text{factor}^L$ (same compounding as §2.4). $<1$ → activations vanish over depth; $>1$ → explode. Same recursion for backward gradients (layer Jacobian $=W$). So preserve variance → forward signal & backward gradient stay $O(1)$ through depth.

Recursion (gives 1/fan_in = Xavier): $z_i=\sum_j W_{ij}x_j$, $W$ & $x$ i.i.d. mean-0, independent → variances add:

$$\text{Var}(z) = \text{fan\_in}\cdot\text{Var}(W)\cdot\text{Var}(x)$$

Preserve → $\text{Var}(W)=1/\text{fan\_in}$ → $\text{std}=1/\sqrt{\text{fan\_in}}$ (assumes linear / tanh-near-0).

Where the 2 comes from (He, ReLU): ReLU $\max(0,z)$ zeros the negative half, so the second moment passed forward is halved:

$$E[a^2]=E[\max(0,z)^2]=\tfrac12 E[z^2]=\tfrac12\text{Var}(z)$$

Redo: $\text{Var}(z^l)=\text{fan\_in}\cdot\text{Var}(W)\cdot\tfrac12\text{Var}(z^{l-1})$; preserve → $\text{Var}(W)=2/\text{fan\_in}$ → $\text{std}=\sqrt{2/\text{fan\_in}}$.

The 2 exactly cancels ReLU's ½. Use Xavier's 1/fan_in on a ReLU net and the signal shrinks ~½ each layer → vanishes over depth. He doubles the weight scale to compensate for the half ReLU kills.

📄 Related: Day 1 §3.1; Q13 (compounding over depth).

Q16 — What is fan_in?

Definition: fan_in = number of inputs feeding into a neuron (input dimension; how many weighted connections get summed for one output). fan_out = number of outgoing connections (output dimension).
  • Linear 512→256 ($W$ is 256×512): $\text{fan\_in}=512$, $\text{fan\_out}=256$.
  • Conv: $\text{fan\_in}=\text{in\_channels}\times k_h\times k_w$; $\text{fan\_out}=\text{out\_channels}\times k_h\times k_w$.

Why it appears in init: a neuron sums fan_in products, so $\text{Var}(z)=\text{fan\_in}\cdot\text{Var}(W)\cdot\text{Var}(x)$ — output variance grows linearly with fan_in. Summing 1000 inputs has ~100× the variance of summing 10. To keep scale constant, weight variance $\propto 1/\text{fan\_in}$ (Xavier) or $2/\text{fan\_in}$ (He). More inputs → bigger sum → smaller weights.

📄 Related: Day 1 §3.1; Q15 (variance recursion).

Q17 — Why scale residual branches by 1/√N_layers

Setup: a residual block adds into a running stream $x_l = x_{l-1} + F_l(x_{l-1})$, so the stream is a sum over all $N$ blocks. Variances add → if each contributes $\approx\sigma^2$: $\text{Var}(x_N)\approx\text{Var}(x_0)+N\sigma^2$ — variance grows linearly with depth (std ~ $\sqrt N$).

That means late-layer activations dwarf early ones → instability, and each new block's relative contribution shrinks like $1/l$.

Fix: scale each branch by $\alpha$: $x_l = x_{l-1} + \alpha F_l(x_{l-1})$ → $\text{Var}(x_N)\approx\text{Var}(x_0)+N\alpha^2\sigma^2$. Set $\alpha=1/\sqrt N$:

$$N\cdot\tfrac1N\sigma^2 = \sigma^2 \quad\text{(constant, depth-independent)}$$

Why √N not 1/N: variances add, not std. A sum of $N$ independent terms has variance $\propto N$ → divide std by $\sqrt N$. Same $1/\sqrt{\text{count}}$ pattern as $1/\sqrt{\text{fan\_in}}$ (fan_in normalizes the sum within a layer; $1/\sqrt N$ normalizes the sum across depth). Used by GPT-2 init; DeepNorm / Fixup / T-Fixup do depth-dependent residual scaling.

📄 Related: Day 1 §3.2; Q16 (fan_in), Q15 (variance preservation).

Q18 — What are LN/RMSNorm, and why are BN stats noisy?

LayerNorm (per example, across the $d$ features): $\mu=\tfrac1d\sum_i x_i$, $\sigma^2=\tfrac1d\sum_i(x_i-\mu)^2$, $\hat{x}_i=\tfrac{x_i-\mu}{\sqrt{\sigma^2+\epsilon}}$, $y_i=\gamma_i\hat{x}_i+\beta_i$.

RMSNorm (drop the mean): $y_i = \gamma_i\,\dfrac{x_i}{\text{RMS}(x)}$, $\text{RMS}(x)=\sqrt{\tfrac1d\sum_i x_i^2+\epsilon}$ — cheaper, ~equal quality. BatchNorm (per feature, over the batch): $\mu_i=\tfrac1B\sum_b x_{b,i}$, $\sigma_i^2$ over $B$ examples.

Why BN stats are noisy: $\mu_i,\sigma_i^2$ are sample estimates from only $B$ examples. SE of the mean $\propto 1/\sqrt B$ (variance estimate noisier) → with small $B$ they jump batch-to-batch → an example's normalization depends on its random batch-mates (coupling + noise). Worse for transformers: small effective batch (long sequences), ragged variable-length batches, $B=1$ decoding (stats undefined).

LN/RMSNorm use the $d$ features of ONE example ($d$ large, fixed) → batch-independent, deterministic, identical at train and inference. (BN uses batch stats at train but a running EMA at test → a different function in each mode → a classic bug source.)

📄 Related: Day 1 §4.1–4.2; Q15 (variance), Q16 (fan_in).

Q19 — Pre-norm vs post-norm (why pre-norm is stable)

Residual in practice: ResNet $y = x + F(x)$ ($F$ = conv-bn-relu-conv-bn); transformer wraps attention & FFN in residuals.

Equations (sublayer $S$): $$\text{Post-norm: } x_{l+1}=\text{LN}(x_l + S(x_l)) \qquad \text{Pre-norm: } x_{l+1}=x_l + S(\text{LN}(x_l))$$

def postnorm(x): x = ln1(x + attn(x)); x = ln2(x + ffn(x)); return x   # LN on trunk
def prenorm(x):  x = x + attn(ln1(x)); x = x + ffn(ln2(x)); return x   # LN on branch
# pre-norm: one final ln_final before the head

Why pre-norm cleaner/stabler: unroll pre-norm $x_L = x_0 + \sum_l S_l(\text{LN}(x_l))$ — a pure additive identity path with nothing on it (LN on the branch) → clean $\times1$ gradient highway at every depth. Post-norm puts LN on the trunk, so the backward gradient crosses $L$ LayerNorm Jacobians ($\neq$ identity) → clean path broken → gradients grow/shrink with depth → needs warmup, unstable deep (Xiong et al. 2020).

Trade-off: pre-norm's clean trunk → residual stream variance grows with depth (§3.2, fixed by $1/\sqrt N$ + final LN). Post-norm keeps activations bounded but gradients fragile. LLMs pick pre-norm: trainability > bounded activations.

📄 Related: Day 1 §4.3–4.4; Q17 (residual variance), Q18 (LN).

Q20 — Why does RMSNorm match LayerNorm? (theory or empirical)

Both. LN = re-center (−μ) + re-scale (/σ); RMSNorm = re-scale only (/RMS).

Theory (Zhang & Sennrich 2019 hypothesis): LN's benefit comes mainly from re-scaling invariance, not re-centering. Normalization helps optimization by controlling activation/gradient magnitude (bounds them, smooths the landscape, stabilizes per-layer effective LR) — that's the /RMS part. Mean subtraction is near-redundant because (a) the learned affine + next linear layer absorb a constant offset, (b) in high dimensions the mean is small relative to scale.

Scale-invariance argument: both LN & RMSNorm satisfy $f(\alpha x)=f(x)$ → self-stabilizing gradient (roughly orthogonal to the weight direction, bounded). RMSNorm keeps this; it only drops the less-important shift-invariance.

Verdict: well-motivated hypothesis + strong empirical confirmation (RMSNorm matches LN, ~7–64% faster; LLaMA/T5/Gemma use it). Not a theorem, but more than "just empirical."

Q21 — Optimizers in detail + how AdamW runs in production

Ladder ($g=\nabla L$): Momentum $v=\beta v+g$, $\theta\mathrel{-}=\eta v$ (accumulate consistent dirs, damp oscillation). RMSProp $s=\rho s+(1-\rho)g^2$, $\theta\mathrel{-}=\eta g/(\sqrt s+\epsilon)$ (per-param adaptive LR). Adam = both + bias-correct. AdamW = Adam + decoupled decay.

$$m=\beta_1 m+(1-\beta_1)g,\ \ v=\beta_2 v+(1-\beta_2)g^2,\ \ \hat m=\tfrac{m}{1-\beta_1^t},\ \hat v=\tfrac{v}{1-\beta_2^t},\ \ \theta\mathrel{-}=\eta\tfrac{\hat m}{\sqrt{\hat v}+\epsilon}+\eta\lambda\theta$$

(bias correction because $m,v$ start at 0; AdamW subtracts $\eta\lambda\theta$ directly from the weights).

Production transformers: AdamW default; $\beta_1=0.9$, $\beta_2=0.95$ (faster 2nd moment, robust to spikes), $\epsilon=10^{-8}$. LR linear warmup → cosine decay to ~10% peak (~1–6e-4). Weight decay ~0.1 decoupled, only on matmul weights (exclude bias / LN-RMSNorm gains / embeddings). Clip global norm 1.0. Memory: $m+v$ per param = 2× params; mixed precision ≈ 16 bytes/param → 7B ≈ 112 GB of states → why FSDP/ZeRO shard optimizer state. Alts: Adafactor (factorized 2nd moment), Lion (sign-based), Shampoo (preconditioned), Muon.

Q22 — How is fuzzy dedup done? What is Jaccard? (MinHash + LSH)

Jaccard = set-overlap similarity of two docs' n-gram shingle sets. MinHash compresses each set to a fixed signature whose match-rate is an unbiased Jaccard estimate; LSH banding turns the all-pairs search into a bucket lookup so you only verify a shortlist.

The goal — Jaccard similarity. Each doc → a set of n-gram shingles (slide an n-token window, keep distinct shingles). Similarity is the overlap fraction:

$$J(A,B)=\frac{|A\cap B|}{|A\cup B|}\in[0,1]\qquad 1=\text{identical},\ 0=\text{disjoint}.$$

Example: $A=\{$the cat sat, cat sat on, sat on mat$\}$, $B=\{$the cat sat, cat sat on, sat on rug$\}$ → $2/4=\mathbf{0.5}$.

Why not compute $J$ directly? Billions of docs → all-pairs is $O(N^2)$ and each set is huge. Two cheats:

1. MinHash — estimate $J$ from $k$ numbers. With random hash $h$, $\text{minhash}_h(A)=\min_{x\in A}h(x)$. The min over $A\cup B$ lands in the intersection iff it's in both sets, so $\Pr[\text{minhash}(A)=\text{minhash}(B)]=J(A,B)$. Use $k$ hashes → length-$k$ signature; the fraction of matching positions is an unbiased estimate $\hat J$ with $\operatorname{SE}=\sqrt{J(1-J)/k}$. $k\approx128$ → any doc is 128 ints, $J$ known to a few %.

2. LSH banding — only compare likely matches. Split the signature into $b$ bands of $r$ rows ($k=b\cdot r$). Two docs are candidate pairs if they collide in any band:

$$\Pr[\text{candidate}]=1-\big(1-s^{\,r}\big)^{b}\qquad\text{S-curve, cliff near } t\approx(1/b)^{1/r}.$$

Tune $(b,r)$ so the cliff sits at the dedup threshold (e.g. drop $J\gtrsim0.8$): pairs above are almost always caught, below almost never become candidates → run exact-Jaccard verification on a tiny shortlist, not all $N^2$ pairs.

Pipeline: shingle → MinHash signature → band → bucket → candidate pairs → verify exact $J$ → drop. Knobs: n (shingle size), k (estimate precision), $(b,r)$ (cliff location). Same three ideas power datasketch, Spark MinHashLSH, and the FineWeb/SlimPajama dedup stacks.
